Implementation Logic

// Calculate remaining (can be negative when over budget)
const remaining = {
  calories: macros.calories.target - macros.calories.current,
  protein: macros.protein.target - macros.protein.current,
}

// Check if over budget
const isOverCalories = remaining.calories < 0
const isOverProtein = remaining.protein < 0

// Display logic:
{isOverProtein ? (
  {Math.abs(remaining.protein)}g protein over
) : (
  {remaining.protein}g protein
)}
{' '}·{' '}
{isOverCalories ? (
  {Math.abs(remaining.calories)} kcal over
) : (
  {remaining.calories} kcal
)} {isOverCalories || isOverProtein ? '' : 'remaining'}

Key Changes Made

  • Removed Math.max(0, ...) wrapper that prevented negative values
  • Added isOverCalories and isOverProtein flags
  • Conditional rendering: red text with "over" suffix when negative
  • Uses Math.abs() to show positive numbers with "over" label
  • Omits "remaining" text when any macro is over budget
